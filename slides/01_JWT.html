JWT - Jason Web Tokens

What is JWT?

JSON Web Tokens are an open, industry standard RFC 7519 method for representing claims securely between two parties.

jwt.io

What does this mean?

  • Compact
  • Self-Contained
  • Information-Container
  • Communication between two parties

→ Nothing more than a JSON-Object, which can be trusted to not have been modified since it was created.

Why would we need that?

  • Authentication
  • Information exchange
  • ...

Structure of a JWT

Tokens consist of three parts separated by . (dots):

  • Header
  • Payload
  • Signature

→ xxx.yyy.zzz

Header

Normally consists of two parts:

  • Algorithm (alg)
  • Token Type (typ) → "JWT"
                    
                        {
                            "alg": "HS256",
                            "typ": "JWT"
                        }

Header (2)

  • Base64Url encoded
  • Builds first part of the token

Payload

  • Contains claims
    • Statements about an entity (e.g. user)
    • Registered
    • Public
    • Private

Payload (2)

A payload could look like this:

                    
                        {
                            "sub": "1234567890",
                            "name": "John Doe",
                            "admin": true
                        }

→ Builds second part of the token

Signature

  • Third part of the token
  • Created by signing xxx.yyy
  • Using a algorithm and a secret

Complete token

→ Debugger

How does it work?

For the authentication use case:

  • Server issues JWT which is stored at client
  • Client sends token in header for every request
                    
                        Authorization: Bearer <token>

Flow

Advantages

  • No session
  • Stateless
  • No database queries
  • Multiple end points possible

JWT compared to SWT & SAML

  • Much less verbose compared to SAML (XML)
  • Possibility to use Private/Public Key pair
  • Easier to parse (JSON vs. XML)

Implementations

  • Libraries for all major programming languages
  • Even implemented in frameworks like Spring Security
